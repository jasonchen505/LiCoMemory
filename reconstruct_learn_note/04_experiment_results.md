## 4. 问题诊断

### 4.1 实体提取失败

#### 症状
- 提取实体数为0
- 提取关系数为0
- 图为空

#### 诊断步骤
1. 检查LLM输出日志
2. 查看提示词内容
3. 验证输出格式
4. 检查解析逻辑

#### 可能原因
1. **LLM输出格式错误**
   - 输出不是JSON格式
   - 输出包含额外文本
   - 输出格式不一致

2. **提示词问题**
   - 指令不清晰
   - 缺少示例
   - 格式要求不明确

3. **模型能力不足**
   - 模型太小
   - 模型不适合此任务
   - 模型参数设置不当

4. **解析逻辑错误**
   - JSON解析失败
   - 字段名不匹配
   - 数据类型错误

#### 解决方案
1. **优化提示词**
   ```
   请从以下对话中提取实体和关系，以JSON格式返回：
   {
     "entities": [{"name": "实体名", "type": "类型"}],
     "relationships": [{"source": "源实体", "target": "目标实体", "relation": "关系"}]
   }
   
   对话内容：
   {dialogue}
   ```

2. **使用更强模型**
   - 使用更大的模型（如Qwen3-14B）
   - 使用指令调优模型
   - 使用专门的实体提取模型

3. **增加后处理**
   - JSON修复
   - 格式验证
   - 错误重试

4. **调试和日志**
   - 保存LLM原始输出
   - 记录解析过程
   - 添加详细日志

### 4.2 评估器配置错误

#### 症状
