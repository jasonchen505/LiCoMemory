# LiCoMemory 技术架构学习笔记

## 1. 整体架构

LiCoMemory是一个基于LLM和知识图谱的长期记忆框架，主要包含以下组件：

### 核心组件
1. **GraphRAG**: 主控制器
2. **GraphRAGCore**: 核心图谱管理
3. **DynamicMemory**: 动态记忆图
4. **QueryProcessor**: 查询处理器
5. **Evaluator**: 评估器

### 数据流
```
输入文档 → 分块 → 实体提取 → 图构建 → 存储
                                    ↓
用户查询 → 查询理解 → 检索 → 重排序 → 答案生成
```

## 2. 图构建流程

### 2.1 会话摘要生成 (Stage 1)
**目的**: 为每个对话会话生成简洁摘要

**流程**:
1. 遍历所有会话
2. 使用LLM生成摘要
3. 保存摘要到文件

**提示词示例**:
```
请为以下对话生成一个简洁的摘要，包含关键信息：
[对话内容]
```

**关键参数**:
- `enable_summary`: 是否启用摘要生成
- `max_concurrent`: 最大并发数

### 2.2 文档分块 (Chunking)
**目的**: 将长文档分割成适合处理的小块

**分块策略**:
- 按token数量分块
- 支持对话格式的特殊处理
- 保留会话元数据

**关键参数**:
- `chunk_token_size`: 分块大小（默认1200 tokens）
- `chunk_overlap_token_size`: 重叠大小（默认100 tokens）
- `dialogue_input`: 是否为对话格式

### 2.3 实体关系提取 (Stage 2)
**目的**: 从每个chunk中提取实体和关系

**流程**:
1. 遍历所有chunks
2. 使用LLM提取实体和关系
3. 解析LLM输出
4. 去重和合并

**提示词设计**:
- 针对对话格式优化
- 支持多种实体类型
- 支持多种关系类型

**关键参数**:
- `entity_merge_threshold`: 实体合并阈值（默认0.85）
- `relationship_merge_threshold`: 关系合并阈值（默认0.9）

### 2.4 图构建
**目的**: 将提取的实体和关系构建成图

**图结构**:
- 节点：实体
- 边：关系
- 属性：时间戳、会话ID等

**使用NetworkX**:
```python
import networkx as nx
G = nx.DiGraph()
G.add_node(entity_id, **properties)
G.add_edge(source_id, target_id, **relation_properties)
```

## 3. 查询流程

### 3.1 查询理解
**目的**: 理解用户查询意图

**处理**:
- 提取查询关键词
- 识别查询类型
- 时间信息提取

### 3.2 检索
**目的**: 从图中检索相关信息

**检索策略**:
1. **实体检索**: 找到与查询相关的实体
2. **关系检索**: 找到相关三元组
3. **会话摘要检索**: 找到相关会话

**相似度计算**:
- 使用Embedding计算语义相似度
- 支持批量计算优化

### 3.3 重排序
**目的**: 对检索结果进行重排序

**CogniRank算法**:
- 实体相似度权重
- 会话相似度权重
- 时间相似度权重
- 加权融合

**关键参数**:
- `enable_CogniRank`: 是否启用CogniRank
- `summary_weight`: 摘要权重
- `rerank_k`: 重排序参数

### 3.4 答案生成
**目的**: 基于检索到的上下文生成答案

**流程**:
1. 构建提示词
2. 包含检索到的三元组和文本
3. 调用LLM生成答案

**提示词模板**:
```
基于以下信息回答问题：
Triples: {triples}
Text Chunks: {chunks}
Question: {question}
Answer:
```

## 4. 评估系统

### 4.1 LLM评估
**方法**: 使用LLM判断答案是否正确

**流程**:
1. 比较模型输出和标准答案
2. 使用LLM判断语义是否一致
3. 统计准确率

**优点**:
- 支持语义匹配
- 处理表述差异

### 4.2 精确匹配评估
**方法**: 字符串精确匹配

**流程**:
1. 标准化字符串
2. 精确比较
3. 统计准确率

**优点**:
- 快速
- 无额外开销

### 4.3 会话匹配评估
**方法**: 评估检索到的会话是否正确

**指标**:
- 检索会话与标准答案会话的匹配度
- 平均匹配分数
